## рџЏ—пёЏ Architecture

The app follows a feature-first architecture:
- `lib/features/`: domain-specific features (e.g., `candles`, `prayers`, `auth`)
- `lib/core/`: shared utilities, services (PayPal, Supabase), and generic widgets
- `lib/widgets/`: App-level shared widgets (MainScaffold, AppDrawer)

### Data Sync
Models are synchronized with the web application (`apps/web`) to ensure data consistency across platforms.
- **Prayers**: Fetched from `Prayer` table.
- **Saints**: Fetched from `Saint` table (logic matches web `feastMonth`/`feastDayOfMonth` query).
