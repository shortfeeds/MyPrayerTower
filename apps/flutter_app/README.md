# MyPrayerTower Mobile App

The official mobile application for MyPrayerTower, built with Flutter. A comprehensive Catholic companion app featuring prayers, virtual candles, saints, bible reading, and more.

## 📱 Features

### 🙏 Prayer & Spirituality
- **Prayer Library**: Extensive collection of Catholic prayers (Rosary, Novenas, Litanies, etc.) organized by category.
- **Prayer Wall**: Community prayer requests with real-time sync. Post requests and pray for others.
- **Daily Reading**: Daily Mass readings and gospel.
- **Bible Reader**: Immersive bible reading experience.
- **Rosary**: Interactive virtual Rosary with bead tracking.

### 🕯️ Virtual Sacramentals
- **Light a Candle**: Light virtual candles with realistic flame animations (`RepaintBoundary` optimized).
- **Mass Offerings**: Request masses for intentions.
- **Memorials**: Create permanent memorials for loved ones.

### 📚 Knowledge & Saints
- **Saint of the Day**: Daily saint bios and prayers, synced with the liturgical calendar.
- **Saints Library**: Searchable database of saints.

### 🛠️ Key Technical Features
- **State Management**: flutter_riverpod
- **Navigation**: go_router with deep linking support
- **Backend**: Supabase (PostgreSQL) for real-time data and authentication
- **Serialization**: Freezed & json_serializable
- **Payments**: PayPal integration for digital goods (Mass offerings, Candles)
- **Ads**: AdMob integration (Banner, Native, Rewarded) with smart placement.

## 🏗️ Architecture

The app follows a feature-first architecture:
- `lib/features/`: domain-specific features (e.g., `candles`, `prayers`, `auth`)
- `lib/core/`: shared utilities, services (PayPal, Supabase), and generic widgets
- `lib/widgets/`: App-level shared widgets (MainScaffold, AppDrawer)

### Data Sync
Models are synchronized with the web application (`apps/web`) to ensure data consistency across platforms.
- **Prayers**: Fetched from `Prayer` table.
- **Saints**: Fetched from `Saint` table (logic matches web `feastMonth`/`feastDayOfMonth` query).

## 🚀 Getting Started

1.  **Prerequisites**:
    - Flutter SDK (Latest Stable)
    - Dart SDK

2.  **Installation**:
    ```bash
    flutter pub get
    ```

3.  **Code Generation** (if modifying models):
    ```bash
    dart run build_runner build --delete-conflicting-outputs
    ```

4.  **Running**:
    ```bash
    flutter run
    ```

## 📦 Building for Release

To build the Android App Bundle (AAB):
```bash
flutter build appbundle
```
*Note: Ensure your `key.properties` is configured for signing.*
